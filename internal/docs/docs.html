Overview

O Bounce e um proxy de dados desportivos que agrega informacao da FPB (www.fpb.pt) e do TugaBasket (resultados.tugabasket.com).

Base URL: http://localhost:3001 (local) ou https://bounce.dribly.pt (producao)

Rate limit: 100 pedidos/minuto por IP

Cache: SQLite com TTL inteligente (jogos do dia: 2min, historico: 24h)

Formatos: JSON em todos os endpoints

Authentication & Rate Limits

A API e publica — nao requer autenticacao.

Rate limit bypass: Envia o header X-Dribly-Key com o valor da variavel de ambiente DRIBLY_KEY.

curl -H "X-Dribly-Key: ${DRIBLY_KEY}" http://localhost:3001/api/games?club=119&season=2025/2026

WebSocket
WS/ws/game/{id}
Recebe atualizacoes em tempo real durante um jogo.
idstringInternal game ID413420

Eventos:

  • score_update — score mudou
  • game_started — jogo comecou
  • game_finished — jogo terminou
{"type":"score_update","data":{"id":"413420","equipa_casa":"SL Benfica","resultado_casa":89,"resultado_fora":73,"equipa_fora":"FC Porto"}}

Self-Hosting

Docker:

docker run -d --name bounce --restart unless-stopped \
  -p 3001:3001 -v bounce-data:/data \
  ghcr.io/mefrraz/bounce:latest

Go binary:

git clone https://github.com/mefrraz/bounce.git
cd bounce
go build -o bounce ./cmd/server
BOUNCE_DATA_DIR=/data ./bounce

Variaveis de ambiente:

BOUNCE_PORTPorta HTTP (default: 3001)3001
BOUNCE_DATA_DIRDiretoria para SQLite (default: /data)/data
DRIBLY_KEYChave para bypass do rate limit-
